# Live-Demo: Feature → Test → Fix

- **Feature ändern** (UI, klein & sichtbar)
- **Tests reagieren** (E2E / QA schlägt an)
- **Fix mit AI** (Tests wieder grün)

::: notes
Kontext (≈15s):
„Wir ändern jetzt live ein kleines UI-Verhalten.
Normalerweise zieht das Tests und QA nach sich.“

Aktion (≈30s):
– Kleine UI-Änderung (z. B. Label oder Feld)
– Tests laufen / schlagen an
– Fix mit Junie, ohne manuelles Suchen

Einordnung (≈15s):
„AI kennt den Code-Kontext.
Ich entscheide – AI setzt um.“

Fallback:
– Falls Tests nicht laufen: Diff + Test-Code zeigen
– Falls Zeit knapp: direkt zur Einordnung springen
:::
