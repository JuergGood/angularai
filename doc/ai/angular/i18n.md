# Internationalization & Help System Proposal

## 1. Internationalization (i18n)

### 1.1 Implementation Strategy
We will use **ngx-translate** for dynamic language switching in the Angular frontend. This allows us to switch languages without page reloads, which is essential for the requested "language switcher menu".

#### Key Components:
- **ngx-translate/core**: The core library for translations.
- **ngx-translate/http-loader**: To load translation files via HTTP.
- **I18nService**: A custom wrapper service using Angular Signals to manage the current language state.

### 1.2 Localization Files
Translations will be stored in `src/assets/i18n/`:
- `en.json`: English translations.
- `de-ch.json`: German (Switzerland) translations.

### 1.3 Usage in Components
- **Templates**: Use the `translate` pipe: `{{ 'DASHBOARD.TITLE' | translate }}`
- **Code**: Use `translateService.get()` or `instant()`.

## 3. Help System

### 3.1 Static Page Generation
We will use a script to convert the existing Markdown files into a JSON-based structure or static HTML fragments that can be rendered within the Angular app.

#### Source Files:
- `README.md`
- `doc/userguide/user-guide.md`
- `doc/userguide/admin-guide.md`
- `doc/userguide/faq.md`

#### Rendering:
- Create a `HelpContentComponent` that takes a page ID and renders the corresponding content.
- Update `app.routes.ts` to include a path for help pages: `/help/:pageId`.

### 3.2 Navigation
Add a "Help" submenu in the Config menu containing:
- README
- User Guide
- Admin Guide (Visible to Admins only)
- FAQ
